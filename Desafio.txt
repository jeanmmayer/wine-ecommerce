Uma antiga loja de vinhos gostaria de vender seus produtos também pela internet para aumentar o seu faturamento, com isso,
o seu objetivo é desenvolver um pequeno e-commerce web que contemple os seguintes requisitos funcionais:

    1) Cadastro de vinho com as informações de nome, tipo do vinho e peso;
    2) Registrar uma venda, considerando as seguintes situações:
    3) Cliente gostaria de selecionar um ou mais produtos e a quantidade desejada;
    4) Informa a distancia para entrega (ex: 250km)
    5) Ao fechar o pedido, deve ser informado o total geral da venda, considerando o frete.

Fórmula para o cálculo do frete:

    1) Para cada 1 kilo do total do pedido, considerar R$5 no calculo do frente;
    2) Distancias até 100km considerar somente valor calculado do peso do pedido,
    distancias maiores que 100km considerar valor calculado do peso do pedido vezes distancia da entrega divido por 100.

Não é necessário controlar o estoque dos produtos;

    Requisitos
    1) Ter a camada de frontend totalmente desacoplada do backend;
    2) Utilizar o conceito REST, não utilizar framework.
    3) Não usar frameworks de HTML/CSS (como bootstrap)
    4) Utilizar PHP ou Java para o desenvolvimento do backend
    5) Não há problemas em usar frameworks/libs JS que estão em evidência no mercado (usamos React atualmente)

Entrega do desafio:

    1) Enviar projeto em um repositorio no git;
    2) Informar requisitos necessários para o funcionamento da aplicação (ex: servidor web, banco de dados, etc);
    3) Informar configurações necessárias para o funcionamento da aplicação (ex: criação de banco, definição de senha de acesso ao banco, etc);"
    4) Informar em um descritivo, a explicação (arquitetura) da aplicação desenvolvida, o que motivou a escolha de cada API/framework/lib
    no projeto e diferenciais que queira detalhar. Se precisar de mais detalhes, é só perguntar.
